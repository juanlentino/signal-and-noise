# v9.3.0 — Long-form post layout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship theme v9.3.0 — the long-form post layout minor for single-note posts: drop caps + footnotes (brutalist styling on `core/footnotes` + hover-popover JS) + Tufte-style sidenotes via CSS float-right + frontmatter spec card replacing the existing minimal meta group.

**Architecture:** All four components scoped to single-note posts (`templates/single.html`). Pure-CSS components (drop caps, footnote styling, sidenote styling, frontmatter CSS) append to `assets/css/critical.css` per v9.2.0 precedent. The only PHP with logic is the pillar shortcode (`inc/post-frontmatter.php`) — TDD'd. The footnote popover JS is a progressive enhancement loaded conditionally on `is_singular('post')`. The frontmatter template part REPLACES (not extends) the existing `.sn-note-meta` group in `templates/single.html`.

**Tech Stack:** PHP 8.0+, WordPress 6.4+, vanilla JS (no JSX build step), block themes (FSE), standalone PHP test runner (`php tests/<file>.php`).

**Spec:** `docs/superpowers/specs/2026-05-26-v9.3.0-long-form-post-layout-design.md` (commit `93ed82e`)

**Plan-phase decisions locked beyond spec §5:**

| # | Decision | Rationale |
|---|---|---|
| D7 | CSS additions append to `assets/css/critical.css` | v9.2.0 precedent (pull-quote/steps-enumerated/compare-columns/post-closing all went there); inlined critical CSS avoids FOUC |
| D8 | `inc/post-frontmatter.php` loaded via explicit `require_once` in `functions.php` | Established pattern: functions.php currently requires 11 inc/ modules in sequence |
| D9 | `assets/js/footnotes-popover.js` enqueued conditionally on `is_singular('post')` | Only load where it might be used |
| D10 | Frontmatter template part REPLACES the existing `.sn-note-meta` group in `templates/single.html` | The existing group already occupies the "frontmatter" position above post-title with date + reading-time; the spec card extends it with TAGS + PILLAR rather than duplicating placement |
| D11 | Popover JS uses safe DOM manipulation (cloneNode + appendChild), NOT innerHTML | Security hygiene: even though source is WP-sanitized post content, avoiding innerHTML eliminates the XSS surface entirely |

```php
<?php
/**
 * Signal & Noise — Post frontmatter pillar shortcode.
 *
 * Registers [sn_post_pillar] — used inside parts/post-frontmatter.html
 * to render the PILLAR slot when a post is tagged with a pillar slug.
 *
 * Convention-based tag matching. The shortcode degrades gracefully:
 * posts whose tags don't match any pillar return empty string.
 *
 * @package SignalNoise
 * @since 9.3.0
 */

if ( ! defined( 'ABSPATH' ) ) {
	exit;
}

function sn_post_pillar_shortcode() {
	$pillar_map = array(
		'provenance' => array(
			'label' => 'PROVENANCE',
			'href'  => '/provenance/over-detection/',
		),
		// Add additional pillars here as their essay URLs are published.
	);

	$post = get_post();
	if ( ! $post ) {
		return '';
	}

	$tags = wp_get_post_tags( $post->ID, array( 'fields' => 'slugs' ) );
	if ( empty( $tags ) || ! is_array( $tags ) ) {
		return '';
	}

	foreach ( $tags as $tag_slug ) {
		if ( isset( $pillar_map[ $tag_slug ] ) ) {
			$p = $pillar_map[ $tag_slug ];
			return sprintf(
				'<a class="sn-post-frontmatter__pillar" href="%s">%s</a>',
				esc_url( $p['href'] ),
				esc_html( $p['label'] )
			);
		}
	}

	return '';
}
add_shortcode( 'sn_post_pillar', 'sn_post_pillar_shortcode' );
```

### Step 5.1: Create the popover JS

- [ ] **Create `assets/js/footnotes-popover.js`** — uses safe DOM manipulation (cloneNode + appendChild), NOT innerHTML:

```javascript
/**
 * Signal & Noise — Footnote hover-popover.
 *
 * Progressive enhancement on core/footnotes (WP 6.3+). Footnotes work
 * without this JS via WP's default scroll-to-footnote-list behavior.
 * This module adds a hover-preview popover so readers don't lose their
 * reading position.
 *
 * Security: uses safe DOM cloning (cloneNode + appendChild) — never
 * innerHTML. Source content (the footnote li) is already-sanitized WP
 * post_content, but we avoid innerHTML to eliminate the XSS surface.
 *
 * Mobile / coarse pointer: skipped entirely. Tap on sup falls back to
 * default scroll behavior.
 *
 * @since theme v9.3.0
 */
( function () {
	'use strict';

	if ( window.matchMedia && window.matchMedia( '(pointer: coarse)' ).matches ) {
		return;
	}

	var activePopover = null;

	function buildPopover( anchorLink ) {
		var href = anchorLink.getAttribute( 'href' );
		if ( ! href || href.charAt( 0 ) !== '#' ) {
			return null;
		}
		var targetId = href.slice( 1 );
		var targetLi = document.getElementById( targetId );
		if ( ! targetLi ) {
			return null;
		}

		var popover = document.createElement( 'div' );
		popover.className = 'sn-footnote-popover';

		// Safe DOM clone: iterate child nodes, cloneNode each, appendChild.
		// Skip back-link anchors (we're at the source). NO innerHTML.
		var children = Array.prototype.slice.call( targetLi.childNodes );
		for ( var i = 0; i < children.length; i++ ) {
			var node = children[ i ].cloneNode( true );
			if ( node.nodeType === 1
				&& node.tagName === 'A'
				&& node.getAttribute( 'href' )
				&& node.getAttribute( 'href' ).indexOf( '#footnote-ref-' ) === 0 ) {
				continue;
			}
			popover.appendChild( node );
		}

		return popover;
	}

	function positionPopover( popover, anchorRect ) {
		document.body.appendChild( popover );
		var popoverRect = popover.getBoundingClientRect();
		var spaceBelow = window.innerHeight - anchorRect.bottom;
		var top;
		if ( spaceBelow >= popoverRect.height + 16 ) {
			top = anchorRect.bottom + window.scrollY + 4;
		} else {
			top = anchorRect.top + window.scrollY - popoverRect.height - 4;
		}
		var left = anchorRect.left + window.scrollX;
		var maxLeft = window.scrollX + window.innerWidth - popoverRect.width - 8;
		if ( left > maxLeft ) { left = maxLeft; }
		if ( left < 8 ) { left = 8; }
		popover.style.top = top + 'px';
		popover.style.left = left + 'px';
	}

	function removeActive() {
		if ( activePopover && activePopover.parentNode ) {
			activePopover.parentNode.removeChild( activePopover );
		}
		activePopover = null;
	}

	function onEnter( event ) {
		var sup = event.target.closest( 'sup' );
		if ( ! sup ) { return; }
		var anchor = sup.querySelector( 'a[href^="#footnote-"]' );
		if ( ! anchor ) { return; }
		removeActive();
		var popover = buildPopover( anchor );
		if ( ! popover ) { return; }
		activePopover = popover;
		positionPopover( popover, sup.getBoundingClientRect() );
		popover.addEventListener( 'pointerleave', removeActive );
	}

	function onLeave( event ) {
		var sup = event.target.closest( 'sup' );
		if ( ! sup ) { return; }
		var relatedTarget = event.relatedTarget;
		if ( activePopover && relatedTarget && activePopover.contains( relatedTarget ) ) {
			return;
		}
		removeActive();
	}

	function init() {
		document.addEventListener( 'pointerenter', onEnter, true );
		document.addEventListener( 'pointerleave', onLeave, true );
	}

	if ( document.readyState === 'loading' ) {
		document.addEventListener( 'DOMContentLoaded', init );
	} else {
		init();
	}
} )();
```
