# Theme v9.6.0 Implementation Plan — Prep Minor for v10.0.0

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the theme-side additive prep work for v10.0.0 — register one missing diagnostic ability, add a WP 7.0 admin notice as the user-facing warning, ship a CHANGELOG entry that announces the v10.0.0 plan.

**Architecture:** Mirrors `inc/abilities-*.php` module convention. One new file (`inc/admin-notice-wp-version.php`) for the WP-version notice (self-contained — deleted in v10.0.0 after WP 7.0 hard-raise). One ability added to the existing `inc/abilities-diagnostics.php` module.

**Reference spec:** [`docs/superpowers/specs/2026-05-27-v5-and-v10-paired-cycle-design.md`](../specs/2026-05-27-v5-and-v10-paired-cycle-design.md) — §3 v9.6.0 prep-minor scope.

**Sequencing prerequisite:** v4.6.0 plugin ship + UAT stable (per spec §1). This plan does not begin execution until v4.6.0 + post-ship audit + any v4.6.x patches are gated.

## Scope context — Audit findings that revised the spec estimate

The spec §3 estimated ~280 LOC for v9.6.0 based on "register 5–8 missing abilities + WP 7.0 notice + @deprecated annotations." Live audit of the theme (commit `23a15e9`, v9.5.0) found:

1. **12 abilities already registered** (per `inc/abilities-registration.php` docblock + verified via grep). The theme is more formalized than the spec assumed.
2. **Genuine gaps inventory:**
   - `sn_gh_latest_theme_tag()` (in `inc/wp-update-integration.php`) — returns latest GitHub release tag; useful for AI agents checking if a theme update is available. **NO ability replacement.** → Add `signal-and-noise/get-latest-theme-tag`.
   - `sn_purge_all_caches()` / `sn_clear_template_overrides()` (in `inc/template-maintenance.php`) — these are theme-side helpers called via cross-package contract filters. The PLUGIN's `signal-noise/purge-all-caches` and `signal-noise/clear-template-overrides` abilities already dispatch them. Adding theme-side duplicate abilities (`signal-and-noise/purge-all-caches`) would create parallel surface with no new behavior. **SKIP — duplicate surface.**
3. **No `@deprecated` annotation work.** Theme's `v10.0.0-scope.md` (committed during v9.5.0) audited all 79 public surface items and found **0 candidates for removal/rename**. There is no theme-side non-Ability surface to deprecate. The corresponding spec task is empty in practice.
4. **Theme has no admin pages.** The WP 7.0 admin notice renders via the standard `admin_notices` hook on all wp-admin pages (the same hook the plugin uses).

**Revised LOC estimate: ~200 LOC.** Smaller than spec's 280 but still substantive (1 new ability + admin notice + tests + CHANGELOG). The "@deprecated annotations" workstream is correctly empty because the theme is already canonically Ability-formalized.

This delta is normal — plans audit reality; specs sketch direction. The strategic shape (theme prep minor announcing v10.0.0) is unchanged.

```php
/**
 * Ability execute_callback for signal-and-noise/get-latest-theme-tag.
 *
 * Delegates to sn_gh_latest_theme_tag() in inc/wp-update-integration.php.
 * That function handles the GitHub API call + caching + filter dispatch
 * to the plugin (via sn_gh_latest_theme_tag_result contract listener).
 *
 * @param mixed $input { force_refresh?: bool } or null.
 * @return array{ok:bool,tag:?string}
 */
function sn_theme_ability_get_latest_theme_tag( $input ) {
	if ( ! function_exists( 'sn_gh_latest_theme_tag' ) ) {
		return array( 'ok' => false, 'tag' => null );
	}
	$force_refresh = is_array( $input ) && ! empty( $input['force_refresh'] );
	$tag = sn_gh_latest_theme_tag( $force_refresh );
	if ( ! is_string( $tag ) || '' === $tag ) {
		return array( 'ok' => false, 'tag' => null );
	}
	return array( 'ok' => true, 'tag' => $tag );
}
```

## Task 2: Create `inc/admin-notice-wp-version.php` — WP < 7.0 dismissible admin notice

**Why this task:** the user-facing warning that v10.0.0 will require WP 7.0. Mirrors the plugin's v4.6.0 admin-notice file (same architectural pattern; different namespace + dismissal key).

**Files:**
- Create: `inc/admin-notice-wp-version.php`
- Modify: `functions.php` (1 require_once line)

- [ ] **Step 1: Create the file**

```php
<?php
/**
 * Signal & Noise theme — WP version pre-warning admin notice (v9.6.0).
 *
 * Dismissible admin notice rendered on every wp-admin page when the
 * current WP version is < 7.0. Announces that v10.0.0 will require WP 7.0.
 *
 * Persisted via user-meta key `sn_theme_dismissed_wp_version_notice_v960`
 * so each admin user can dismiss independently. The version suffix allows
 * future minors to re-introduce the notice if needed.
 *
 * v10.0.0 hard-raises Requires at least: 7.0, after which this file is
 * deleted entirely (along with its require_once in functions.php).
 *
 * Mirrors the plugin's inc/admin-notice-wp-version.php pattern (same
 * v4.6.0 cycle). Kept theme-namespaced to avoid collision when both are
 * loaded on the same WP install.
 *
 * @package SignalAndNoise
 * @since 9.6.0
 */

if ( ! defined( 'ABSPATH' ) ) {
	exit;
}

/**
 * Test seam: returns the current WordPress version.
 *
 * Wraps global $wp_version so tests can stub it. Test stub override:
 * if $GLOBALS['SN_THEME_WP_VERSION_OVERRIDE'] is non-empty, returns it.
 * (Global, not constant — constants are immutable and tests need to
 * exercise both <7.0 and >=7.0 paths in a single run.)
 *
 * @return string e.g. "6.4.2" or "7.0".
 */
function sn_theme_get_wp_version() {
	if ( ! empty( $GLOBALS['SN_THEME_WP_VERSION_OVERRIDE'] ) ) {
		return (string) $GLOBALS['SN_THEME_WP_VERSION_OVERRIDE'];
	}
	global $wp_version;
	return is_string( $wp_version ) ? $wp_version : '0.0';
}

/**
 * Renders the WP < 7.0 admin notice if applicable.
 *
 * Hooked to admin_notices. Bails when:
 *   - WP version is >= 7.0
 *   - Current user has dismissed via user-meta sentinel
 *   - User lacks manage_options
 */
function sn_theme_render_wp_version_notice() {
	if ( ! current_user_can( 'manage_options' ) ) {
		return;
	}
	if ( version_compare( sn_theme_get_wp_version(), '7.0', '>=' ) ) {
		return;
	}
	$user_id = get_current_user_id();
	if ( $user_id && get_user_meta( $user_id, 'sn_theme_dismissed_wp_version_notice_v960', true ) ) {
		return;
	}

	$dismiss_url = wp_nonce_url(
		add_query_arg( 'sn_theme_dismiss_wp_version_notice', '1' ),
		'sn_theme_dismiss_wp_version_notice'
	);

	printf(
		'<div class="notice notice-warning is-dismissible"><p><strong>Signal &amp; Noise theme:</strong> v10.0.0 will require WordPress 7.0 (you are on %s). Plan your upgrade — the v10.0.0 release will refuse to load on WP &lt; 7.0.</p><p><a href="%s">Dismiss this notice</a></p></div>',
		esc_html( sn_theme_get_wp_version() ),
		esc_url( $dismiss_url )
	);
}
add_action( 'admin_notices', 'sn_theme_render_wp_version_notice' );

/**
 * Handles dismiss-link click — records user-meta sentinel.
 */
function sn_theme_handle_wp_version_notice_dismiss() {
	if ( empty( $_GET['sn_theme_dismiss_wp_version_notice'] ) ) {
		return;
	}
	if ( ! current_user_can( 'manage_options' ) ) {
		return;
	}
	check_admin_referer( 'sn_theme_dismiss_wp_version_notice' );

	update_user_meta( get_current_user_id(), 'sn_theme_dismissed_wp_version_notice_v960', '1' );

	wp_safe_redirect( remove_query_arg( array( 'sn_theme_dismiss_wp_version_notice', '_wpnonce' ) ) );
	exit;
}
add_action( 'admin_init', 'sn_theme_handle_wp_version_notice_dismiss' );
```
