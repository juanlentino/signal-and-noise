## Source-grounded design decisions

These reference the actual `wp-admin/includes/plugin.php` behavior. Each is also documented in WORDPRESS-REFERENCE.md gotchas #14–#17.

### 1. Submenu registration order (gotcha #14)
The first `add_submenu_page( $parent, …, $menu_slug, … )` call where `$menu_slug === $parent_slug` overrides the auto-prepended "duplicate parent" entry. This is the documented workaround for the `Signal & Noise / Signal & Noise` sidebar bug. Plugin v1.8.1 already does this — keep it. After the override, register the remaining 7 submenu items in display order (Identity, Login, Cloudflare, Plausible, RSS, Reading Time, Links).

### 2. Hook suffix collection (gotcha #15)
`add_menu_page()` always returns a string. `add_submenu_page()` returns `false` if user lacks capability. The enqueue guard collects all returned hooks, filters out `false`, and matches via `in_array( $hook, $sn_hooks, true )`. This guards against the silent-break path where a future non-admin user role gets `false` and the stylesheet stops loading.

### 3. Slug uniqueness (gotcha #16)
Each submenu gets a unique slug. The display-order array is the single source of truth for both registration and dispatch. Never call `add_submenu_page` from inside a hook that can fire twice (it'd silently duplicate entries). `admin_menu` fires exactly once per request — safe.

### 4. Form processing pattern (gotcha #17)
Keep the existing inline `$_POST` processing at the top of `sn_theme_options_page()`. It runs before any output, so it's functionally equivalent to a `load-{hookname}` handler. Switching to `load-` is only worth it if we wanted Post/Redirect/Get (we don't — the current pattern surfaces save status via `$notices` rendered in the same request, which feels snappier).

### 5. URL backward compat
Old v1.8.x URLs use `admin.php?page=sn-theme-options&tab=identity`. New v1.9.0 URLs use `admin.php?page=sn-identity` (no `tab` param). Both must work. The dispatcher logic:
```
if $_GET['tab'] is in $valid_tabs: use it (legacy v1.8.x path)
else: derive tab from $_GET['page'] via $slug_to_tab map
else: default to 'dashboard'
```

```php
/**
 * The 8 SN admin pages, each rendered by sn_theme_options_page().
 *
 * Defined once at module scope so registration and dispatch read from
 * a single source of truth. Slug uniqueness is critical — WP's
 * add_submenu_page() has no duplicate detection (gotcha #16 in
 * docs/WORDPRESS-REFERENCE.md), so a typo here would silently produce
 * a phantom sidebar entry.
 *
 * Order in the array = display order in the WP sidebar.
 */
function sn_admin_pages() {
	// Note: the 'dashboard' slug ('sn-theme-options') intentionally matches
	// the parent menu slug to suppress WP's auto-prepended duplicate-parent
	// submenu entry (gotcha #14). Order matters: must be first submenu
	// registered.
	return array(
		array( 'slug' => 'sn-theme-options', 'tab' => 'dashboard',    'label' => 'Dashboard',     'title' => 'Signal & Noise — Dashboard' ),
		array( 'slug' => 'sn-identity',      'tab' => 'identity',     'label' => 'Identity',      'title' => 'Signal & Noise — Identity' ),
		array( 'slug' => 'sn-login',         'tab' => 'login',        'label' => 'Login',         'title' => 'Signal & Noise — Login' ),
		array( 'slug' => 'sn-cloudflare',    'tab' => 'cloudflare',   'label' => 'Cloudflare',    'title' => 'Signal & Noise — Cloudflare' ),
		array( 'slug' => 'sn-plausible',     'tab' => 'plausible',    'label' => 'Plausible',     'title' => 'Signal & Noise — Plausible' ),
		array( 'slug' => 'sn-rss',           'tab' => 'rss',          'label' => 'RSS',           'title' => 'Signal & Noise — RSS' ),
		array( 'slug' => 'sn-reading-time',  'tab' => 'reading-time', 'label' => 'Reading Time',  'title' => 'Signal & Noise — Reading Time' ),
		array( 'slug' => 'sn-links',         'tab' => 'links',        'label' => 'Links',         'title' => 'Signal & Noise — Links' ),
	);
}

/**
 * Map an admin-page slug to a tab name. Used by sn_theme_options_page()
 * to dispatch when $_GET['tab'] isn't present (v1.9.0+ deep links).
 */
function sn_admin_page_tab_for_slug( $slug ) {
	foreach ( sn_admin_pages() as $page ) {
		if ( $page['slug'] === $slug ) {
			return $page['tab'];
		}
	}
	return 'dashboard';
}

/**
 * Cache of all registered hook suffixes for the SN admin pages.
 * Used by the enqueue guard to load the stylesheet on any of our
 * pages without re-deriving hook names from slugs.
 *
 * add_menu_page() always returns a string; add_submenu_page() returns
 * false when the user lacks the required capability (gotcha #15), so
 * we filter the array before comparing.
 */
function sn_admin_page_hooks( $set = null ) {
	static $hooks = array();
	if ( is_array( $set ) ) {
		$hooks = array_values( array_filter( $set, 'is_string' ) );
	}
	return $hooks;
}

add_action( 'admin_menu', function() {
	$hooks = array();

	$hooks[] = add_menu_page(
		'Signal & Noise',
		'Signal & Noise',
		'manage_options',
		'sn-theme-options',
		'sn_theme_options_page',
		'dashicons-megaphone',
		81
	);

	foreach ( sn_admin_pages() as $page ) {
		$hooks[] = add_submenu_page(
			'sn-theme-options',
			$page['title'],
			$page['label'],
			'manage_options',
			$page['slug'],
			'sn_theme_options_page'
		);
	}

	sn_admin_page_hooks( $hooks );
} );

/**
 * Enqueue the SN admin stylesheet on any of our 8 pages.
 *
 * Guards via in_array() against the collected hook list so a slug
 * rename in sn_admin_pages() won't silently break the guard. Cache-
 * busted by SNT_VERSION.
 */
add_action( 'admin_enqueue_scripts', function( $hook ) {
	if ( in_array( $hook, sn_admin_page_hooks(), true ) ) {
		wp_enqueue_style(
			'sn-admin',
			SNT_URL . 'assets/admin.css',
			array(),
			SNT_VERSION
		);
	}
} );
```

## Rollback paths

**If sidebar shows duplicate "Signal & Noise / Signal & Noise" entries:**
- Cause: the first `add_submenu_page` call doesn't have its slug matching the parent slug (`'sn-theme-options'`). Gotcha #14.
- Fix: verify `sn_admin_pages()` returns the dashboard entry first AND with `slug => 'sn-theme-options'`.

**If a submenu item is missing for a non-admin user (capability lockout):**
- Cause: `add_submenu_page` returned `false`. Gotcha #15.
- Expected behavior — the user lacks `manage_options`. Not a bug.

**If a submenu appears twice in the sidebar:**
- Cause: somehow registered twice. Gotcha #16.
- Fix: `grep -n 'add_submenu_page' inc/admin-page.php` should show exactly ONE call (inside the loop). If two, remove the orphan.

**If the new URL `admin.php?page=sn-identity` shows the Dashboard instead of Identity:**
- Cause: the dispatcher's slug-to-tab map is missing the entry. Or the `$valid_tabs` array doesn't include `'identity'`.
- Fix: verify `sn_admin_page_tab_for_slug()` returns `'identity'` for the slug `'sn-identity'`, and `$valid_tabs` contains `'identity'`.

**If the Login tab shows DORMANT but should be ACTIVE:**
- Cause: `wps-hide-login` plugin is still active. Either deactivate it (recommended — that's the Phase 13 cutover that v1.5.0 absorbed) or accept the dormant state.
- Note: the spec calls this state ACTIVE *only when our module is the one running*. If wps-hide-login is still installed for whatever reason, the status correctly reflects dormancy.
