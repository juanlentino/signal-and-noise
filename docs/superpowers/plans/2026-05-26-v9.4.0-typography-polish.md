# v9.4.0 — Typography polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship typography polish (justified text + hyphenation + hanging punctuation) for body paragraphs in single-note `/notes/<slug>/` posts as theme v9.4.0 — direct sequel to v9.3.0's editorial-spread layout. Pure CSS, no JS, no PHP changes, no new files.

**Architecture:** Three CSS rules appended to [`assets/css/critical.css`](../../assets/css/critical.css) in a new `/* v9.4.0 */` section directly after the existing v9.3.0 long-form post layout section (around line 819). All rules scoped via `.single-post .wp-block-post-content > p` (direct-child selector excludes pattern-nested paragraphs in pull-quote, sidenote, steps-enumerated, footnote-list, etc.). Justified text and hyphenation are responsively gated `@media (min-width: 1024px)` so mobile cascades to left-aligned. Hanging punctuation is universal (progressive enhancement; Safari-only renders today, Chrome/Firefox ignore silently). Hyphens uses `-webkit-hyphens: auto;` paired with `hyphens: auto;` — Safari still requires the prefix as of 2026-05.

**Tech Stack:** Pure CSS3, WordPress 6.4+ (FSE), browser support fallback via CSS cascade.

**Spec:** [`docs/superpowers/specs/2026-05-26-v9.4.0-typography-polish-design.md`](../specs/2026-05-26-v9.4.0-typography-polish-design.md) (commit `b19ff7f`).

**LOC delta vs spec:** spec estimated ~30-50 LOC of CSS. Plan estimates ~50 LOC including section header comments and the explanatory `/* Why ... */` annotations. CHANGELOG entry adds another ~40 LOC. Total ~90 LOC.

## Task 2: Add v9.4.0 typography section to critical.css

### Step 2.1: Insert the v9.4.0 section

- [ ] **Find the closing line of the v9.3.0 section** (located in Task 1). It will be either a `}` at the end of a rule, or possibly already a section break comment.

- [ ] **Insert the following block directly after the last v9.3.0 rule:**

```css

/* ============================================
   v9.4.0 — Typography polish (justified + hyphenation + hanging punctuation)
   ============================================ */

/* Body-paragraph typography for single-note posts.
   Scoped via `> p` direct-child to exclude paragraphs nested in
   patterns (pull-quote, sidenote, steps-enumerated list items,
   footnote list items, code blocks). Those keep their own alignment. */

/* --- JUSTIFIED TEXT + HYPHENATION ---
   Gated @media (min-width: 1024px) — below that, body paragraphs
   cascade to left-aligned (the WP default) to avoid ragged spacing
   on narrow viewports.

   Safari (macOS + iOS) still requires the -webkit-hyphens prefix
   as of 2026-05; pair both declarations so all major browsers see
   hyphenation consistently. */

@media (min-width: 1024px) {
	.single-post .wp-block-post-content > p {
		text-align: justify;
		-webkit-hyphens: auto;
		hyphens: auto;
	}
}

/* --- HANGING PUNCTUATION ---
   Not viewport-gated — applies at all widths. Safari renders
   correctly (full support since Safari 10, 2017). Chrome/Firefox/
   Edge ignore the property as of 2026-05; punctuation stays
   inside the text edge (graceful degradation, no fallback rule
   needed). */

.single-post .wp-block-post-content {
	hanging-punctuation: first last allow-end;
}
```
