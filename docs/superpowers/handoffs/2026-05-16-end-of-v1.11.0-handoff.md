## How to install v1.10.2 + v1.11.0 (testing the new flow)

1. Open `https://juanlentino.com/wp-admin/update-core.php`
2. If "Signal & Noise Tools" doesn't show v1.10.2 or v1.11.0 yet, click "Check Again" (12h cache rollover; otherwise force-check works)
3. Should see "v1.11.0 available" with an Update Now button
4. Click "Update Now"
5. WP downloads from `https://github.com/juanlentino/signal-and-noise-tools/archive/refs/tags/v1.11.0.zip`
6. `upgrader_source_selection` filter renames the unpacked dir from `signal-and-noise-tools-1.11.0/` to `signal-and-noise-tools/`
7. WP installs → plugin re-activates → live

**If it fails**, fall back via:
```bash
gh workflow run deploy.yml --ref v1.11.0 --repo juanlentino/signal-and-noise-tools
```

## The full architectural picture now

### Update flow (post-v1.10.1)
- Tag push → no auto-deploy
- WP polls GitHub Tags API every 12h → finds new tag → puts it in update transient
- User visits `wp-admin → Updates` → sees badge → clicks Update Now → WP downloads + installs
- Emergency hotfix path: `gh workflow run deploy.yml --ref vX.Y.Z` (manual workflow_dispatch trigger)

### Per-post settings (post-v1.10.2)
Six fields in the "Signal & Noise" meta box on post + page editor (auto-converts to block editor sidebar panel):
- `_sn_noindex` (bool) → robots `noindex,nofollow`
- `_sn_noarchive` (bool) → robots `noarchive`
- `_sn_noimageindex` (bool) → robots `noimageindex`
- `_sn_meta_description` (string) → overrides excerpt for `<meta>`, OG, Twitter, JSON-LD
- `_sn_canonical_url` (URL) → overrides `<link rel="canonical">`
- `_sn_og_image_url` (URL) → overrides featured image / auto card / site default

All six exposed via REST at `/wp-json/wp/v2/posts/{id}` → `meta._sn_*`.

### Sitemap (post-v1.11.0)
- TSF currently serves `/sitemap.xml` (live)
- WP core's `/wp-sitemap.xml` is 404 (TSF deregisters it)
- Our `wp_sitemaps_posts_query_args` filter is dormant (no hook firing)
- Phase 13 cutover (v2.0.0): TSF deactivated → WP core sitemap activates → our filter excludes noindex/canonical-overridden posts automatically

## Architectural invariants (preserve in future work)

1. **All form POSTs route through `sn_handle_admin_post()`** in `inc/admin-page.php` (PRG handler). Per-post save handler in `inc/post-settings.php` is the only exception (uses `save_post` hook + its own nonce).
2. **Tag pushes don't auto-deploy** anymore. Manual workflow_dispatch is the emergency-hotfix path. Default flow is WP UI.
3. **Design system classes live in `assets/admin.css`** with CSS vars. Don't reintroduce inline styles even for "just this one thing".
4. **Cross-module function calls use `function_exists()` guards** — defensive against module deactivation. Pattern established in v1.10.0's reader integrations + reused throughout.
5. **`register_post_meta()` with `show_in_rest=true`** for any new per-post meta. Future-proofs for React sidebar migration without storage changes.
