## What's queued next

### Theme v8.5.2 — mirror plugin v1.11.1 cache improvements

The plugin shipped v1.11.1 between when the previous handoff was written (v1.11.0) and now. Its improvements aren't yet in the theme:

1. **Drop cache TTL 12h → 1h** in `inc/wp-update-integration.php`:
   ```php
   const SN_GH_THEME_CACHE_TTL = HOUR_IN_SECONDS; // v8.5.2: 12h → 1h
   ```
   12h is too long for the "I just pushed a tag, where's my update?" cycle.

2. **Honor WP's "Check Again" button** — accept `WP_FORCE_UPDATE_CHECK` constant + `?force-check=1` query arg, pass through to `sn_gh_latest_theme_tag( $force_refresh = true )` which bypasses the transient. Without this, `?force-check=1` returns the same stale cached value.

3. **Version-change cache invalidation** on `admin_init`:
   ```php
   add_action( 'admin_init', function() {
       $last_seen = (string) get_option( SN_GH_THEME_LAST_SEEN_OPT, '' );
       $current   = (string) wp_get_theme( SN_GH_THEME_STYLESHEET )->get( 'Version' );
       if ( $current && $last_seen !== $current ) {
           delete_site_transient( SN_GH_THEME_CACHE_KEY );
           delete_site_transient( 'update_themes' );
           update_option( SN_GH_THEME_LAST_SEEN_OPT, $current );
       }
   } );
   ```
   Handles the "WP UI install just completed, but the update transient still says update available" race.

Estimated work: ~20 min. Reference: [plugin's `inc/wp-update-integration.php` lines 36, 50-57, 115-117, 200-213](https://github.com/juanlentino/signal-and-noise-tools/blob/main/inc/wp-update-integration.php).

## Architectural invariants (preserve in future work)

1. **All form POSTs route through `sn_handle_admin_post()`** in `inc/admin-page.php` (PRG handler). Per-post save handler in `inc/post-settings.php` is the only exception (`save_post` hook + own nonce).
2. **Tag pushes don't auto-deploy** anymore — for *either* repo. Manual `workflow_dispatch` is the emergency-hotfix path. Default flow is WP UI.
3. **Design system classes live in `assets/admin.css`** with CSS vars. No inline styles.
4. **Cross-module function calls use `function_exists()` guards** — defensive against module deactivation.
5. **`register_post_meta()` with `show_in_rest=true`** for any new per-post meta. Future-proofs for React sidebar migration.
6. **WP core primitive integration: read the source first.** Used this session — verified `upgrader_source_selection` against `wp-admin/includes/class-wp-upgrader.php` before shipping. Pattern documented in CLAUDE.md.
