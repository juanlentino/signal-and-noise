## Deferred findings — prioritized

### Tier A — High-value, ship next session (~1 session of work)

These are the biggest remaining wins. Pick one or two per ship.

#### U-03 — Extract modal inline styles to CSS classes
- **Effort:** medium-large (~1h, regression risk on modal layout)
- **Why:** `openApplyModal`, `renderVerdictSuggestion`, `buildDriftModalContent`, `buildOrphanDeleteModalContent`, `renderSuggestion` use ~38 `setAttribute('style', ...)` calls in `assets/health-suggest-actions.js`. `assets/admin.css` has zero rules for `.snt-modal-*` / `.snt-suggest-*` / `.snt-verdict-*`. Same problem in the new `assets/snt-confirm.js` (~12 more inline-style calls).
- **Strategy:** add `.snt-modal-backdrop`, `.snt-modal-box`, `.snt-modal-header`, `.snt-modal-body`, `.snt-modal-pane-before`, `.snt-modal-pane-after`, `.snt-modal-footer`, `.snt-suggest-panel`, `.snt-verdict-panel`, `.snt-confirm-backdrop`, `.snt-confirm-box` to `admin.css`. JS sets `className`. Keep only genuinely-dynamic values inline (`grid-template-columns` that switches on `isMobile`).
- **Verification:** walk the modal at desktop + mobile viewports; verify Delete button doesn't shift; check focus ring and z-index unchanged.
- **Suggested version:** v4.1.2 alone OR bundled with U-15.

#### B-11 — File-size split for the 2 worst offenders
- **Effort:** large (~1.5h, real refactor risk)
- **Why:** Per `CLAUDE.md` "No file over ~150 lines." 13 plugin files violate. The two worst:
  - `inc/abilities-registration.php` — **1642 lines (11×)** — natural split: per-feature ability files (`abilities-ai.php` for the AI abilities, `abilities-system.php` for cache/template/update, `abilities-cron.php` for cron, etc.) with a thin orchestrator that requires them.
  - `inc/admin-page.php` — **1371 lines (9×)** — natural split: extract per-tab POST handlers (currently all in `sn_handle_admin_post`) into per-tab `inc/admin-handlers-*.php` files.
- **Strategy:** abilities-registration first (simpler — pure registration code with no shared state). Then admin-page (trickier — POST handler routing).
- **Test coverage required:** abilities-integration test (157/0) MUST stay green through both splits.
- **Suggested version:** v4.1.2 abilities split, v4.1.3 admin-page split (separate ships to limit blast radius).
