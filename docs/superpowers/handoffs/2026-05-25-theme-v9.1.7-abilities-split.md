## What's queued: theme v9.1.7

### Scope

Mirror the plugin v4.1.3 split structure for theme's `inc/abilities-registration.php` (1814 lines). The theme registers fewer abilities than the plugin (12 per the architecture memory, in 3 categories: `diagnostics`, `content`, `ai-generation`), so the split files will be fewer and smaller.

### Strategy

1. **Read theme `inc/abilities-registration.php` fully** before any structural decisions — per hard-rule memory, never refactor from a snippet.
2. **Identify natural per-feature boundaries.** Theme abilities tend to be design-system-aware (block patterns, design tokens, reading time, brand voice rewrite) per the architecture memory. Likely split shape:
   - `abilities-permission-helpers.php` (named perm callables; mirror plugin pattern)
   - `abilities-categories.php` (the 3 theme categories with `wp_has_ability_category()` guards)
   - `abilities-diagnostics.php` (design-system inspection abilities)
   - `abilities-content.php` (per-post content abilities)
   - `abilities-ai-generation.php` (AI-backed text abilities)
3. **Bootstrap require:** locate where `functions.php` (or another theme entry point) requires `inc/abilities-registration.php` — keep that filename so the orchestrator is a drop-in.
4. **No PHP test suite theme-side.** Verification budget is lower than plugin's. Mitigation: structural mirroring (same patterns as plugin v4.1.3) + manual smoke-walk in wp-admin after deploy.

### Manual smoke-walk (post-deploy)

The theme has no `tests/` directory analogous to the plugin's `abilities-integration.php`. After v9.1.7 lands via wp-admin Updates, walk:

1. **wp-admin command palette ⌘K** — type "design tokens" or "reading time"; the abilities should appear if registered.
2. **REST endpoint** — `curl https://juanlentino.com/wp-json/wp-abilities/v1/abilities/signal-and-noise/<ability-name>` for one ability per split file; expect a 401 (auth required) NOT a 404.
3. **AI Copilot ⌘K dock** — if the theme exposes any `aiCallable: true` abilities, they should appear in the dock surface.

If any of those fail post-deploy, run `gh workflow run deploy.yml --repo juanlentino/signal-and-noise --ref vX.Y.Z` to redeploy. The pre/post-install filter pair (theme v8.5.2+) preserves `.git`, so the manual workflow_dispatch path won't corrupt the working tree.
