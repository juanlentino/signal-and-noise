### v3.5.1 — Nonce conflict + design system alignment hotfix

Caught after user smoke-tested v3.5.0 and reported "AI client not available" / "looks broken" on Webhooks + Health tabs.

Two real bugs, one root cause: I'd shipped v3.4.0 + v3.5.0 without invoking the `superpowers:brainstorming` / `writing-plans` / `verification-before-completion` skills via the Skill tool, and the result was:

1. **Nonce dispatcher conflict** — my new form actions used per-tab nonce names (`sn_health`, `sn_webhooks`); the central `sn_handle_admin_post` in `inc/admin-page.php` calls `check_admin_referer('sn_theme_options_nonce')` before gating on action name, so any unrecognized action dies at `wp_nonce_ays()` ("Link expired") before reaching my handler.
2. **Wrong design system** — I used WP's generic `<table class="form-table">` instead of the bespoke `.sn-fieldset` / `.sn-field` / `.sn-pill` classes that all other SN tabs use (cloudflare-purge.php, plausible-admin.php, etc.).

**Fix:** routed all action dispatch through `sn_handle_admin_post` (the established pattern matches `cf_save`, `pl_save`, etc.); rewrote both renderers using the SN design system. Diagnosed via `superpowers:systematic-debugging`, verified via `superpowers:verification-before-completion`.

This was the discipline-gap turning point. Memory entry [`reference_ai_plugin_v1_features`](../../../.claude/projects/-Users-juanlentino-Projects-signal-and-noise/memory/reference_ai_plugin_v1_features.md) was set after this so future-Claude has the receipts on what `ai/ai` v1.0.0 already does (so we don't duplicate). ### v3.7.0 — Health tab drift detection (IN FLIGHT)

Second net-new AI feature. Adds a 5th check named `drift_time_phrases` to the v3.5.0 Health tab. Detects time-relative phrases ("as of 2024", "recently", "this year", "just released", "the latest", etc.) that decay deterministically as posts age.

**Brainstorm-locked scope (all three picks were "Recommended" defaults):**
- Drift types in v1: time-relative phrases only
- Cost model: per-post AI call, manual trigger only
- Fix flow: detection-only (deep-link to editor; AI-suggested fix deferred to v3.7.x+)

**Hybrid algorithm:** Regex pre-filter (free) eliminates ~80% of posts that have zero candidate phrases; AI second pass evaluates remaining posts' candidates in context vs. the post's `last_modified` date. Only `stale` verdicts surface as findings. Per-scan cost: ~$0.01–$0.02. Annual ceiling at weekly use: under $1.
