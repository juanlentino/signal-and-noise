## Why this session existed

After plugin v1.16.0 shipped the AI Client scaffolding (Phase 12), the open question was: does the **theme** also need pre-7.0 scaffolding? Answer: **no.** The plugin/theme split (Phases 1–5) moved every admin and content-authoring concern out of the theme. The theme has no AI surface to guard.

This handoff captures that audit so future sessions don't re-derive it.

## Why no scaffolding analogue to plugin v1.16.0

The "scaffold now, activate later" pattern only earns its keep when there's a runtime call that needs guarding. Plugin v1.16.0 calls `wp_ai_client_prompt()` — that has to be guarded behind `wp_has_ai_client()` so 6.x doesn't fatal. The theme has no 7.0 API calls of any kind. There's nothing to guard.

## Plugin side — also done

Plugin v1.16.0 ([commit 38ef5e0](https://github.com/juanlentino/signal-and-noise-tools/commit/38ef5e0)) shipped Phase 12 scaffolding:

All `wp_has_ai_client()`-gated. Dormant on 6.x. Activates the moment 7.0 is installed (or 6.x + `wp-ai-client` plugin).

No other pre-7.0 plugin work is queued.
