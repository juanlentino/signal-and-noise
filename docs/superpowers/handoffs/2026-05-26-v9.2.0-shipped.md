## Three primary additions

### 1. Three new block patterns

- **`patterns/pull-quote.php`** — brutalist top + bottom 3px black rules, `#fafafa` bg, Georgia serif italic body, Courier mono small-caps attribution.
- **`patterns/compare-columns.php`** — A vs B two-column layout with vertical 1px black divider via `::before` pseudo on the columns wrapper. Mobile breakpoint hides the divider when columns stack.
- **`patterns/steps-enumerated.php`** — ordered list with monospace zero-padded numerals (`01`, `02`, `03`) via CSS `counter-reset` + `counter-increment` + `::before` pseudo. User adds/removes items; numerals auto-reflow.

All 3 use `Categories: signal-noise` in the docblock — the category was already registered by `inc/patterns.php` since v7.5.0, no new infrastructure needed.

### 2. `post-closing` template part

Auto-rendered on every /notes post via `templates/single.html`'s new template-part include (inserted between `core/post-content` and the existing pillar-link/back-link footer). Contents:

- 1px black separator (top border)
- Tags row (Courier mono small-caps, "Tagged · provenance · music")
- Chronologically-next post link (Courier mono "Next →" label + Georgia serif red-accent title)

Uses `core/post-terms` + `core/post-navigation-link` — zero new PHP, pure block markup + CSS.

### 3. View Transitions catalog → single-note morph

`inc/blocks-view-transitions.php` registers a `render_block_core/post-title` filter that injects `view-transition-name: sn-note-<slug>` into the outer heading/anchor tag. **Same hook covers BOTH catalog cards (rendered inside a `core/query` loop in /notes) AND the single-note article hero** — browser matches transition-names across source and destination, morphs title position from card → hero on navigation.

Why this approach beats per-template editing: block themes use core/query loops for catalog enumeration; emitting per-post inline styles from template markup requires a custom dynamic block. The filter approach is ~50 LOC, single hook, covers every render of `core/post-title` anywhere on the site.

Slug sanitized to CSS-identifier syntax (`[^a-z0-9-]+` collapses to single hyphen, lowercase). Reduced-motion respect inherited from v9.0.0's `@media (prefers-reduced-motion: reduce) { @view-transition { navigation: none; } }` block — per-element morphs auto-disable when navigation is `none`. Mobile carveout (`max-width: 768px`) disables the morph on small screens where the card → hero geometry change feels disorienting.

## Discipline lessons codified this cycle

This cycle reinforced patterns rather than introducing new memory entries:

1. **Visual companion was high-leverage for design questions.** Showing pattern wireframes and in-context article mockups in the browser collapsed several rounds of text Q&A into single visual approvals. Worth defaulting to for any frontend/visual brainstorming work.

2. **Self-review during plan-writing caught a real spec error.** The spec said "create `inc/blocks-pattern-categories.php` (new file)" — but `inc/patterns.php` already registered the `signal-noise` category since v7.5.0. The plan-write phase surfaced this and corrected the spec inline. Reinforces the existing memory: **read the actual code state before assuming what infrastructure exists.**

3. **Security-reminder hook caught dynamic-code-execution in a test.** The initial draft of `tests/patterns-registry.php` parsed `inc/patterns.php` source via a PHP dynamic-execution function (the one the security hook explicitly warns about). Hook blocked the Write. Restructured to use a stubbed `add_action` that immediately invokes the closure with our captured `register_block_pattern_category` stub. Hook just paid for itself.

4. **The implementer's "test baseline is broken" finding (Task 1) was a real catch.** A less-careful implementer would have either (a) ignored the failure or (b) tried to fix the abilities tests, scope-creeping the v9.2.0 work. Surfacing the issue as a v9.2.1 candidate while proceeding with v9.2.0 scope was the right call. Pattern: **pre-existing breakage in unrelated test infrastructure is a flag-and-continue, not a fix-now.**
