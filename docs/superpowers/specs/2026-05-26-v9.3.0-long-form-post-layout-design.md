## 1. Headline

**v9.3.0 is the long-form post layout minor.** Four components layered onto the single-note page transform it from "generic WP post with v9.2.0 patterns" into "printed editorial spread":

1. **Drop caps** on first paragraph (Bebas Neue, blood-red, 3-line drop)
2. **Footnotes** built on `core/footnotes` (brutalist styling + hover-popover JS enhancement)
3. **Sidenotes** (Tufte-style marginalia via CSS float-right at ≥1280px; inline-below at narrow)
4. **Frontmatter spec card** above the post title (DATE / READ TIME / TAGS / PILLAR — mirrors the catalog's `.sn-notes-row-spec` vocabulary)

All four scoped to **single-note posts only** (`templates/single.html`). The /notes catalog page (`inc/page-notes-render.php`) is unchanged. Pillar essays at `/provenance/*` are unchanged.

---

## 2. Why this minor exists

v9.2.0 shipped patterns (pull-quote, compare-columns, steps-enumerated) + the catalog → single-note title morph + the post-closing template part. The single-note page now has solid structural blocks. **But the page-level typography vocabulary is still WP-default**: no drop caps, no proper footnotes (writers fall back to `<sup>1</sup>` workarounds), no marginalia register, no visible metadata above the title.

The brutalist + industrial-catalog + NIN-booklet aesthetic implies these features. Drop caps in Bebas Neue, footnotes numbered like the `01`/`02`/`03` steps pattern, sidenotes in DM Mono small-caps in the margin — every one of these feels inevitable for the brand. v9.3.0 ships them.

**One coherent narrative:** /notes posts now look like a print editorial spread, not a generic blog post.

## 4. Architecture (per component)

### 4.1 Drop caps

Pure CSS. Selector: `.wp-block-post-content > p:first-of-type::first-letter` scoped to `body.single-post` (or equivalent single-template body class).

```css
.single-post .wp-block-post-content > p:first-of-type::first-letter {
    font-family: 'Bebas Neue', Impact, sans-serif;
    font-size: 5rem;
    line-height: 0.85;
    float: left;
    color: var(--wp--preset--color--blood, #e00404);
    margin: 0.1em 0.5rem 0 -0.05em;
    padding-top: 0.3rem;
}
```

**Edge case handling:** `:first-of-type` ensures drop cap only renders when the first child IS a paragraph. Posts opening with a heading, blockquote, or pull-quote pattern skip the drop cap naturally — no PHP conditional needed.

**Reduced motion / accessibility:** drop cap is static (no animation); no reduced-motion concern. Screen readers read the first letter as-is (no `aria-hidden` needed; the float doesn't break reading order).

### 4.2 Footnotes (foundation + styling + popover)

**Foundation: `core/footnotes` block** (WP 6.3+ — theme already requires 6.4+, safe). Writers add it via the inserter; WP auto-handles numbering, references, in-content `<sup>` markers, and end-of-post footnote list.

**Brutalist styling:**

```css
/* In-text <sup> markers */
.wp-block-post-content sup a[href^="#footnote-"] {
    font-family: 'DM Mono', 'Courier New', monospace;
    font-size: 0.75em;
    color: var(--wp--preset--color--blood, #e00404);
    text-decoration: none;
    margin: 0 0.1em;
}

/* End-of-post footnote list */
.wp-block-footnotes {
    border-top: 1px solid var(--wp--preset--color--bone, #000);
    padding-top: 1.5rem;
    margin-top: 3rem;
    counter-reset: sn-footnote;
    list-style: none;
}
.wp-block-footnotes li {
    counter-increment: sn-footnote;
    position: relative;
    padding-left: 3rem;
    font-family: 'DM Mono', 'Courier New', monospace;
    font-size: 0.85rem;
    line-height: 1.5;
    margin-bottom: 0.75rem;
}
.wp-block-footnotes li::before {
    content: counter(sn-footnote, decimal-leading-zero);
    position: absolute;
    left: 0;
    color: var(--wp--preset--color--blood, #e00404);
    letter-spacing: 0.1em;
}
.wp-block-footnotes a[href^="#"] {
    color: var(--wp--preset--color--blood, #e00404);
    text-decoration: none;
}
```

**Hover-popover enhancement (progressive, ~50 LOC vanilla JS):**

`assets/js/footnotes-popover.js`:
- On `pointerenter` of any `sup a[href^="#footnote-"]`:
  - Find the target `<li>` in `.wp-block-footnotes`
  - Build a `<div class="sn-footnote-popover">` containing the li's content
  - Position absolute relative to the sup (below if room, above if not)
  - Append to body
- On `pointerleave` (sup OR popover) → remove
- Mobile (`(pointer: coarse)` media query): listener is a no-op → tap falls back to default scroll-to-footnote behavior
- `prefers-reduced-motion`: no fade-in transition; popover appears instantly

Loaded via `wp_enqueue_script` in `functions.php` (or equivalent), conditional on `is_singular('post')`.

### 4.3 Sidenotes

**Approach: pattern + CSS float-right.** Simplest CSS-only sidenote technique that keeps the sidenote near its authored location.

**Pattern file `patterns/sidenote.php`:**

```php
<?php
/**
 * Title: Sidenote
 * Slug: signal-noise/sidenote
 * Categories: signal-noise
 * Description: Margin annotation at wide viewports (≥1280px); inline below at narrower.
 */
?>
<!-- wp:paragraph {"className":"sn-sidenote"} -->
<p class="sn-sidenote">A brief author commentary that runs alongside the relevant paragraph at wide viewports.</p>
<!-- /wp:paragraph -->
```

**CSS:**

```css
/* Wide viewports: float into right margin */
@media (min-width: 1280px) {
    .sn-sidenote {
        float: right;
        clear: right;
        width: 180px;
        margin-right: -200px;
        margin-top: 0;
        padding-left: 1rem;
        font-family: 'DM Mono', 'Courier New', monospace;
        font-size: 0.75rem;
        line-height: 1.45;
        color: var(--wp--preset--color--rust, #666);
        text-align: left;
    }
}

/* Narrow viewports: inline below with hairline rule */
@media (max-width: 1279.98px) {
    .sn-sidenote {
        margin-top: 0.75rem;
        margin-bottom: 1.5rem;
        padding-top: 0.5rem;
        border-top: 1px solid var(--wp--preset--color--concrete, #d9d9d9);
        font-family: 'DM Mono', 'Courier New', monospace;
        font-size: 0.85rem;
        line-height: 1.5;
        color: var(--wp--preset--color--rust, #666);
    }
}
```

**The `.wp-block-post-content` parent needs no positioning change** — `float: right` works inside the content's normal block-flow without requiring `position: relative` on the parent.

**Edge case — consecutive sidenotes:** `clear: right` enforces stacking, preventing overlap. Long sidenotes accept-the-tradeoff (writer's responsibility to keep brief; the brand voice already favors compression).

**No JS.** Pure CSS-only sidenote technique.

### 4.4 Frontmatter spec card

**Template part `parts/post-frontmatter.html`** included in `templates/single.html` BEFORE the post-title block:

```html
<!-- wp:template-part {"slug":"post-frontmatter","area":"uncategorized"} /-->
<!-- wp:post-title ... -->
```

**Template part content (block markup):**

```html
<!-- wp:group {"className":"sn-post-frontmatter","layout":{"type":"flex","flexWrap":"wrap"}} -->
<div class="wp-block-group sn-post-frontmatter">
  <!-- wp:post-date {"className":"sn-post-frontmatter__date","format":"Y.m.d"} /-->
  <!-- wp:paragraph {"className":"sn-post-frontmatter__rt"} -->
  <p class="sn-post-frontmatter__rt">[sn_reading_time]</p>
  <!-- /wp:paragraph -->
  <!-- wp:post-terms {"term":"post_tag","className":"sn-post-frontmatter__tags","separator":" · "} /-->
  <!-- wp:shortcode -->
  [sn_post_pillar]
  <!-- /wp:shortcode -->
</div>
<!-- /wp:group -->
```

**PHP helper `inc/post-frontmatter.php`:**
- Registers `[sn_post_pillar]` shortcode
- Pillar detection: convention-based — checks post tags for slug matches against the registered pillar slugs (`provenance`, `ai` — extend as pillars grow)
- If matched: renders `<span class="sn-post-frontmatter__pillar"><a href="/provenance/over-detection/">PROVENANCE</a></span>` (link target derives from pillar's canonical essay URL)
- If no match: renders nothing (empty string — degrades gracefully)

```php
/**
 * v9.3.0 pillar map: ONLY pillars with canonical essay URLs ship in the map.
 *
 * As of v9.3.0, the only pillar with a published essay surface is
 * "provenance" (two essays at /provenance/over-detection/ and
 * /provenance/as-substrate/; the map currently links to the canonical
 * over-detection essay). When new pillars are added (e.g., an "ai"
 * pillar essay at /ai/...), add a row here in a v9.3.x patch.
 *
 * The shortcode degrades gracefully: posts tagged with a slug NOT in
 * the map return empty string (the frontmatter spec card simply omits
 * the PILLAR slot for those posts).
 */
function sn_post_pillar_shortcode() {
    $pillar_map = array(
        'provenance' => array( 'label' => 'PROVENANCE', 'href' => '/provenance/over-detection/' ),
        // Add additional pillars here as their essay URLs are published.
    );
    $post = get_post();
    if ( ! $post ) { return ''; }
    $tags = wp_get_post_tags( $post->ID, array( 'fields' => 'slugs' ) );
    foreach ( $tags as $tag_slug ) {
        if ( isset( $pillar_map[ $tag_slug ] ) ) {
            $p = $pillar_map[ $tag_slug ];
            return sprintf(
                '<a class="sn-post-frontmatter__pillar" href="%s">%s</a>',
                esc_url( $p['href'] ),
                esc_html( $p['label'] )
            );
        }
    }
    return '';
}
add_shortcode( 'sn_post_pillar', 'sn_post_pillar_shortcode' );
```

**CSS:** ~40 LOC styling the frontmatter group as a horizontal DM Mono spec row at wide viewports; stacks vertically at narrow.

**Tests `tests/post-frontmatter.php`:** ~60 LOC verifying the shortcode's tag → pillar mapping. Fixtures with various tag combinations; assert correct HTML output or empty string for no-match.

## 5. Architecture decisions (locked from brainstorm)

| # | Decision | Locked | Rationale |
|---|---|---|---|
| D1 | Foundation for footnotes: `core/footnotes` (WP 6.3+) | YES | Auto-numbering + reference management for free. Theme already requires WP 6.4+. Custom impl would duplicate core. |
| D2 | Sidenote positioning: CSS `float: right` (not absolute, not JS-measured) | YES | Float keeps sidenote near authored line position; absolute positioning would lose the Y coordinate without JS measurement. Float-right is the standard CSS-only Tufte technique. |
| D3 | Drop cap selector: `:first-of-type` (not `:first-child`) | YES | `:first-of-type` skips drop cap when post opens with heading/quote — handles edge case naturally without PHP conditional. |
| D4 | Pillar detection: convention-based tag matching (not settings-driven) | YES | 2 pillars today; admin UI is overkill. Tag slug `provenance` → pillar "PROVENANCE"; new pillars add a row to the map. |
| D5 | Popover JS is progressive enhancement (not core dependency) | YES | If popover JS fails to load, footnotes still work via WP default scroll-to-footnote behavior. No JS-only critical path. |
| D6 | Scope: single-note posts only | YES | Catalog and pillar essays already have distinct treatments; this minor doesn't touch them. |

## 6. Risk register

| Risk | Severity | Mitigation |
|---|---|---|
| Drop cap baseline alignment off (Bebas Neue first-letter vs Georgia body) | Medium | Manual UI tuning of `margin-top` + `line-height` during execution; document the magic numbers in the CSS comment so future maintainers know why |
| Sidenote `float: right` overlaps with adjacent floats (e.g., two sidenotes near each other) | Low | `clear: right` enforces stacking; document the writer-side guidance ("keep sidenotes brief") |
| Popover JS conflicts with WP admin-bar or other site JS | Low | Vanilla DOM APIs only; namespaced class (`sn-footnote-popover`); event-delegated listener |
| `core/footnotes` block markup changes in future WP versions, breaking CSS selectors | Low | Selectors target the stable HTML output (`<sup>`, `.wp-block-footnotes`, `<li>`); if WP changes them, easy to update |
| Frontmatter spec card disrupts the existing post-title spacing | Medium | Manual UI tuning during execution; document the spacing tokens; if disruptive, add `margin-bottom` to the frontmatter group |
| Pillar tag map drifts from actual pillar URLs | Medium | Hardcoded map in `inc/post-frontmatter.php`; comment notes "update when pillars change"; test fixture documents the canonical map |
| Long sidenotes push paragraph content awkwardly (white space asymmetry) | Low | Editorial responsibility; v9.3.0 doesn't enforce length limits |
